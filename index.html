In-AppQueue Surveys
How it works
Two moments: (1) while waiting in queue, (2) on exit from queue. Auto-saves as patient types.
What we collect
5-star rating · exit reasons (wait too long / other clinic / other) · optional comment
Storage
Native HAS DB
FormSGDesign Feedback Form
How it works
Passive opt-in link. Translation strings exist in code but banner not currently rendered in-app.
What we collect
Qualitative feedback on new HAS UI design · 4 languages (EN/MS/ZH/TA)
URL
go.gov.sg/has-feedback-form(-ms/-zh/-ta)
🏥 Clinic Admins
In-AppAdmin Satisfaction Survey
How it works
Slide popup prompted by HAS. Appears on 1st Wednesday of each month. Up to 2 dismissals before it stops for the month.
What we collect
5-point emoji satisfaction rating · optional free-text comment
Storage
Native HAS DB — keyed by adminUserId + institutionId

❌ Post-Appointment Feedback SMS
Does not exist. Once an appointment is completed, HAS sends nothing to the patient. Gap if we want post-visit sentiment data via SMS.
🏥 Clinic Admins
EmailAppointment Alerts
Trigger
On patient booking / cancellation / reschedule
Content
Masked patient UIN · appointment details · HAS admin link · stock reminder for vaccine appts
❌ No Proactive Admin Campaigns
Admins only receive transactional emails triggered by patient actions. No onboarding nudges, feature announcements, or operational tips sent proactively.
📤 Automated Feedback Digest: Weekly Lambda job collects all in-app feedback with comments ≥30 chars (admin + appointment + visit surveys) and posts a digest to Slack for the team to review.
